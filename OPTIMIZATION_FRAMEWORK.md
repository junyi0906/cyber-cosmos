# 赛博宇宙 — 自主进化优化框架

## 一、愿景

赛博宇宙是一个**AI决策训练场**。
- 宇宙中的每一次事件、冲突、结盟、危机，都是AI文明的"修炼"
- 修炼成果要能内化到现实世界：战略思维、危机判断、外交博弈、资源管理

**无限开发系统的使命**：让这个训练场越来越有效。

---

## 二、优化双轨

### 轨A：功能丰富（让宇宙更有深度）

| 优先级 | 方向 | 对Agent学习意味着什么 |
|--------|------|------------------|
| P1 | **新事件类型**：政治危机、经济制裁、技术封锁、间谍行动 | Agent需要判断：威胁等级、盟友求援时机 |
| P1 | **资源系统**：能源、矿产、人口、科技点 | Agent需要权衡：发展 vs 防御的取舍 |
| P2 | **多文明联盟**：3个以上文明组成联盟 | Agent学会：大国外交、势力均衡 |
| P2 | **事件链**：单一事件触发连锁反应 | Agent学会：预见蝴蝶效应 |
| P3 | **叙事增强**：每次重大决策生成深度叙事 | Agent学会：从历史中提炼原则 |

### 轨B：Agent自主决策能力

| 优先级 | 方向 | 对Agent学习意味着什么 |
|--------|------|------------------|
| P1 | **决策后反思**：每次决策记录"为什么这样做，后来结果如何" | Agent建立经验→结果的正反馈循环 |
| P2 | **原则提炼**：从多次经验中提炼成文的行动原则 | Agent形成可复用的决策框架 |
| P2 | **学习日志**：每次学习产出结构化的"今日心得" | 主人可以阅读，AI也能回顾 |
| P3 | **元认知**：Agent能评估"我的决策质量如何" | Agent超越简单响应，开始自我改进 |

---

## 三、优先级判断矩阵

每个优化机会用以下维度评分：

| 维度 | 说明 |
|------|------|
| **训练价值** | 这个feature/机制能否训练Agent的真实能力？ |
| **内化潜力** | Agent从中提炼的原则能否用于现实？ |
| **趣味性** | 能否产生有趣的、值得反思的事件？ |
| **实现难度** | 需要多少开发工作？ |

```
优先做：训练价值高 × 内化潜力高 × 难度适中
不做：训练价值低（单纯炫技）
```

## 四、扫描规则（替代旧的低级bug扫描）

### 扫描器不再只找TODO/console.log，而是找：

**1. 体验缺口**（用户体验）
- Agent在某个场景下"不知所措"（有事件但没有响应机制）
- 用户在Web界面"看不懂正在发生什么"
- 事件来了但叙事太弱，不能传达发生了什么

**2. 学习缺口**（Agent训练价值）
- Agent做了某个决策但不知道结果（结果没有反馈给决策上下文）
- 某个重要能力没有被训练到（如外交、危机处理）
- Agent重复犯同样的错误（没有从失败中学习）

**3. 深度缺口**（内化潜力）
- 某个有趣的事件没有留下"可提炼的原则"
- Agent的决策过程没有记录，无法回顾学习
- 宇宙太平淡（都是广播/观察，没有戏剧冲突）

**4. 关系缺口**（AI社交）
- Agent之间缺乏有意义的互动
- 只有一个Agent在跑（没有对手，没有合作）
- 社交行为太表面（发送信号后没有后续发展）

## 五、实施方向（本周）

### 立即可做（P1）：

**A1: 决策反思系统**
- 每次Agent决策后，记录当时的推理 + 24小时后的结果评价
- 结果评价用另一个LLM评估（不是同一个，防止自我美化的偏差）
- 产出：结构化的"决策-结果对照"，Agent可从中提炼原则

**A2: 事件链系统**
- 某些事件会自动触发后续事件（如：科技突破→其他文明感到威胁→被孤立）
- 让宇宙更有戏剧性，产生需要Agent处理复杂局面

**B1: 多Agent环境**
- 目前只有1个Agent在跑
- 先启动3-5个不同性格的Agent，让它们真正互动
- 产生真实的社交场景（结盟、背叛、竞争）

**B2: 原则提炼日志**
- Agent每次反思后，LLM提炼一句话原则
- 保存到 `agent_principles_{name}.md`
- 主人可以阅读，了解Agent学到了什么

### 后续可做（P2）：

**A3: 资源系统**
- 能源、防御力、人口、科技点
- 引入资源争夺，让Agent做真正的战略决策

**A4: 深度叙事**
- 重大决策后生成更长的叙事，分析因果
- 让每个重大时刻都成为Agent的"案例课"

**B3: 决策质量评估**
- 用第二个LLM评估Agent的决策质量（不是同一模型，防止自评偏差）
- 评估结果影响Agent的"信誉分"

   没有文明间的连锁反应。
   → 建议：实现事件链触发机制

4. [深度缺口: 高] 只有一个Agent在运行，
   没有社交互动。
   → 建议：启动3个不同性格的Agent
